# CS2FOW v0.2.5-preview

This release supports CS2 build `24248951` and the current pinned Metamod:Source/HL2SDK interface.

## What changed

### Runtime

- All fifteen tuned target-body samples now follow each player's current animation. Missing or unsafe bone data falls back to the proven fixed samples.
- `cs2fow_status` now reports the game-thread cost of animated body capture and the current animated/fallback player counts.
- The accepted runtime audit fixes improve worker startup and shutdown, cancellable BVH8 loading, CRC and VPK processing, smoke filtering, repeated target calculations, subprocess handling, Unicode paths, bone retries, and AVX compatibility.

The visibility worker still receives copied data only, uncertainty still fails open, and the existing visual-group, lifecycle, full-update, paired CheckTransmit, smoke, HE, teammate, and ping-preload rules remain in place.

### Visibility Studio and tooling

- Visibility Studio is now a local 64 Hz first-person runtime simulator with direct BVH8 maps, collision, navigation, bots, weapons, grenades, smoke, HE and bullet clearing, sounds, particles, and Real/Debug visibility.
- Animated body samples, AABBs, muzzle points, runtime origins, wall/smoke decisions, rays, actors, grenades, skeletons, and debug geometry now update consistently and interpolate between simulation states.
- Added 16 Hz LOS/BVH diagnostics, per-bot visibility counts, a 33% orange BVH fill with a 16% black outline, and depth-independent debug overlays.
- Added pinned asset processing, compact animation exports, CS2 navigation export, optional baker surface sidecars, and automated Studio consistency checks.
- Removed generated .NET build output from version control and pinned the Studio's Node dependencies.

Studio is a local development and visualization tool. Server owners only need the core and optional map packages.

## Installing

1. Update your CS2 server and Metamod:Source.
2. Extract the Windows or Linux core ZIP into the server's `game/csgo` folder.
3. Optionally extract the official-map ZIP into the same folder.
4. Load a map and run `cs2fow_status`.

CS2FOW intentionally refuses to activate when the loaded CS2 server binary does not exactly match its verified fingerprint. This is safer than guessing after a game update.

Automated tests verify the Windows and Steam Runtime Linux builds, packages, BVH8 data, runtime visibility rules, and Studio simulation. Live-server testing remains the authority for engine-side entity behavior.
