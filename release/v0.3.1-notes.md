# CS2FOW 0.3.1

CS2FOW 0.3.1 is a focused visibility-quality update for 0.3.0. It supports the current public CS2 build `24304127` (`1.41.7.2`) on Windows and Linux servers.

## Update from 0.3.0

1. Stop the server.
2. Update the CS2 server and Metamod:Source.
3. If you changed `game/csgo/cfg/cs2fow.cfg`, save a copy of it.
4. Download the 0.3.1 core ZIP for the **server's operating system**: Windows or Linux.
5. Extract the ZIP directly into the server's `game/csgo` folder and allow it to replace the old CS2FOW files.
6. If you keep a customized config, set `cs2fow_visibility_hold_ms 47` to use the new three-tick anti-flicker default.
7. Start the server, load a map, and run `cs2fow_status` in the server console.

The ZIP already contains the correct `addons`, `cfg`, and `tools` folders. After extraction, they must be directly inside `game/csgo`, not inside another nested CS2FOW folder.

No uninstall, client download, database change, or intermediate version is required.

### Official maps

The CS2 build and the 23 validated map files have not changed since 0.3.0. If the 0.3.0 official-map package is installed and working, keep it. New installations can use `cs2fow-0.3.1-official-maps.zip`.

## What changed

- **Fewer missed edge cases.** Eight padded corners around the player's live bounding box are checked again before the full animated hitbox-capsule test. This gives thin visible edges, shoulders, and feet another safe way to reveal the player.
- **Less flicker.** After a player becomes visible, CS2FOW now keeps that result for about three server ticks by default instead of roughly one tick.
- **Cheap checks run first.** CS2FOW tries the previous visible result, chest, bounding-box corners, and weapon muzzle before performing the more expensive capsule test.
- **More honest diagnostics.** The temporary in-game LOS debug view includes the restored bounding-box samples, and Visibility Studio follows the same runtime order and settings.
- **Current build dependencies.** Windows and Steam Runtime 3 Linux packages are built against current Metamod:Source and HL2SDK revisions.

The capsule system, two-worker default, smoke blocking, HE smoke clearing, teammate filtering, automatic map baking, equipment grouping, and fail-open safety model continue to work as before.

## Confirm that it works

Run:

```text
cs2fow_status
```

The first line should say `active` and show the loaded map. If it reports a server fingerprint mismatch, finish updating CS2 and restart the server. If the command is unknown, confirm that Metamod:Source is current and that the ZIP was extracted into the correct folder.
